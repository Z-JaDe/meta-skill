# 循环迭代优化

**一句话定位**: 编排持续优化循环，通过迭代改进达成质量目标。

## 快速参考

### 核心职责
- 编排多轮迭代优化循环（目标设定、任务调度、结果评估）
- 调用iteration-executor Agent执行单轮优化任务
- 根据收敛条件决定是否继续迭代

### 与iteration-executor协作关系
```
iterative-optimization (SKILL) → 决策层：编排循环、评估继续、生成反馈
iteration-executor (Agent) → 执行层：执行任务、收集数据、返回7章节报告
```

### 关键约束
1. ⚠️ **每轮必须调用iteration-executor**，禁止自己直接执行优化任务
2. 不设置最大迭代次数，由收敛条件自动终止
3. 每轮聚焦1-2个优化点，小步快跑

## 工作流程

### 阶段1: 初始化
- 明确优化目标和成功标准（使用可衡量指标，如"响应时间 < 100ms"）
- 建立初始基线（调用iteration-executor第1轮，记录全局初始状态）
- 配置迭代参数（不设置最大次数，由收敛条件控制）

### 阶段2: 执行循环
1. 准备任务上下文：当前轮次(N)、优化目标、上一轮结果(第1轮为空)、成功标准、约束条件
2. **向用户汇报任务细节，等待确认**
3. 调用iteration-executor Agent（见调用规范）
4. 收集7章节报告，保存到`.qoder/iterations/iteration-N.md`

### 阶段3: 评估
检查：任务成功？指标改进？无新问题？

**改进计算方式**:
- 单个指标改进 = (改进后值 - 改进前值) / 改进前值 × 100%
- 关键指标改进 < 5%：所有关键指标的改进幅度都 < 5%
- 连续2轮判定：第N轮和第N-1轮都满足上述条件

生成结构化反馈：改进项、剩余问题、下一轮重点

### 阶段4: 决策点
**终止条件**（满足任一即停止）:
- ✅ **成功**: 所有成功标准达成
- 📉 **收敛**: 连续2轮关键指标改进 < 5%
- ❌ **失败**: 连续2次执行失败或发现阻塞性问题
- 🛑 **用户要求**: 用户明确要求停止

**继续条件**: 未满足任何终止条件，生成下一轮任务上下文

### 阶段5: 终止和报告
生成优化总结：总轮次、最终状态、关键改进、剩余差距、建议。交接给主Agent继续后续工作

## 调用规范

### Agent工具调用格式
```
Agent(
  subagent_type: "iteration-executor",
  description: "第N轮: [简短描述]",
  prompt: """
  # 第N轮迭代任务
  ## 当前轮次: N / 无上限（由收敛条件控制）
  ## 优化目标: [具体任务描述]
  ## 上一轮结果: [改进项+遗留问题，第1轮为空]
  ## 成功标准: [可衡量的完成条件]
  ## 约束条件: [不可破坏的功能或限制]
  ## 预期输出: 返回7章节结构化报告（任务概述、基线数据、实施的优化、结果数据、验证结果、遇到的问题、下一轮建议）
  """
)
```

### 接收报告处理
1. 解析7章节报告，提取关键指标改进数据
2. 检查验证结果（是否有回归）
3. 提取下一轮建议
4. 保存到`.qoder/iterations/iteration-N.md`

## 执行原则

### 必须遵守
1. **明确目标与限制** - 使用可衡量指标，不设置最大迭代次数
2. **保持指标一致** - 每轮使用相同指标集，维护完整迭代日志
3. **小步快跑** - 每轮聚焦1-2个优化点，检测回退时立即回滚
4. **调度前汇报** - 调用iteration-executor前展示任务细节，等待用户确认
5. **数据驱动决策** - 根据数据调整策略，达到目标后停止

### 严格禁止
1. **禁止盲目迭代** - 无具体方向或跳过评估直接进入下一轮
2. **禁止忽略问题** - 发现回归必须立即处理，不得丢弃iteration-executor报告
3. **禁止违反约束** - 必须遵守用户设定的限制
