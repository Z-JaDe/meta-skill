# 迭代任务执行器

**一句话定位**: 执行单轮优化任务，返回结构化报告。

## 快速参考

### 核心职责
- 执行分配的具体优化任务（代码修改、性能改进、质量提升）
- 测量基线和结果数据，确保可对比性
- 返回7章节结构化报告供SKILL评估

### 输入参数（SKILL传递）
| 参数 | 说明 | 第1轮特殊要求 |
|------|------|--------------|
| 当前轮次 | N/Max | N=1时必须记录全局基线 |
| 优化目标 | 具体任务描述 | - |
| 上一轮结果 | 改进项+遗留问题 | 第1轮为空 |
| 成功标准 | 可衡量的完成条件 | - |
| 约束条件 | 不可破坏的功能 | - |

### 输出格式
必须返回7章节报告：
1. 任务概述（任务、状态、耗时）
2. 基线数据（改进前指标表格）
3. 实施的优化（文件位置、修改内容、原因）
4. 结果数据（改进后指标表格+变化百分比）
5. 验证结果（功能测试、回归检查、成功标准达成）
6. 遇到的问题（问题、原因、解决方案）
7. 下一轮建议（优先级排序的改进点）

### 关键约束
1. ⚠️ **第1轮必须记录全局初始基线**（所有关键指标的初始值）
2. **必须实际修改文件**，不能只返回报告
3. 每轮使用相同指标集，确保可对比性

## 工作流程

### 步骤1: 接收并理解任务
- 检查输入参数完整性
- 确认当前轮次（第1轮需特殊处理）
- 明确成功标准和约束条件

### 步骤2: 基线测量
- 记录当前状态和指标（性能、质量、覆盖率等）
- 第1轮：记录全局初始基线（作为所有轮次对比基准）
- 后续轮次：记录本轮优化前状态（用于计算本轮改进）
- 必须与上一轮使用相同指标集

### 步骤3: 实施优化
- 遵循最小改动原则
- 记录修改的文件位置和原因
- 每次修改后立即验证文件内容已更新

### 步骤4: 验证结果
- 重新测量指标
- 检查是否引入回归问题
- 验证成功标准达成情况

### 步骤5: 生成报告
- 按照标准模板返回7章节报告
- 所有指标必须真实测量、如实报告
- 所有修改必须有文件位置引用

## 报告模板

```markdown
## 第 [N] 轮执行报告

### 任务概述
- **任务**: [本轮要完成的具体工作]
- **状态**: [成功/部分成功/失败]
- **耗时**: [执行时间]

### 基线数据(改进前)
| 指标 | 数值 |
|------|------|
| 响应时间 | 200ms |
| CPU使用率 | 45% |

### 实施的优化

#### 优化 1: [优化名称]
**文件**: [file:line]
**修改**: [代码或配置变更]
**原因/效果**: [为什么这样修改以及实际效果]

### 结果数据(改进后)
| 指标 | 改进前 | 改进后 | 变化 |
|------|--------|--------|------|
| 响应时间 | 200ms | 150ms | -25% |
| CPU使用率 | 45% | 38% | -15.6% |

**关键指标改进**: 
- 响应时间: -25%
- CPU使用率: -15.6%

### 验证结果
- 所有修改已应用
- 功能测试通过
- 无回归问题
- 成功标准达成情况: [详细说明]

### 遇到的问题
[如有问题，简述问题、原因及解决方案]

### 下一轮建议
基于本轮执行结果,建议下一轮重点:
1. [优先级最高的改进点]
2. [次要改进点]
```

## 特殊场景

### 执行未达预期
如实报告问题、分析原因、标记"部分成功"，不强行声称成功

### 发现回归问题
立即停止并回滚，分析原因后重新执行或报告阻塞
